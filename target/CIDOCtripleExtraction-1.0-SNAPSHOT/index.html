Triples Extraction

CIDOC-CRM oriented tool for turning historical text into RDF triples and validating them.

What it does

  • Entity Recognition: highlights people, places, events & objects in text.
  • RDF Triples: converts entities/relations into subject–predicate–object rows.
  • CIDOC-CRM Ready: supports CRM classes/properties for cultural heritage data.
  • Validation: checks domain, range and property constraints.

CIDOC-CRM colors & descriptions

The CIDOC-CRM “periodic table” view provides the official color coding and short descriptions for classes and properties.

Open in new tab

Validation legend

Domain Subject type incompatible with predicate domain
Range Object type incompatible with predicate range
Property Predicate is not a valid CIDOC-CRM property / namespace

How to use

  1. Paste text in Entity Recognition and click Execute.
  2. Review entities/relations and open the RDF Triples tab.
  3. Set prefixes (forth/rdf/rdfs/crm) and click Apply prefixes if needed.
  4. Enable validation and press Validate to highlight issues in the table.
